This page provides a full explanation of dashboard

Overview

Dashboard is a project manegment and data visulization tool that allow us to stay update on the network status of all routers, by display in a beautiful and easy way to use.

The main pages

Routers information
Displays all active routers, their software and their subnet.
Discarded Packets
The amount of data that is lost in the network and is classified by service and region .
service : BE, AF1, AF2, AF3, AF4, EF, CS6, CS7.
region : MEN , IPBB.
Utilization
view lists the interfaces that are the most heavily used. The view is a table summary of the interfaces whose utilization exceeds the user-configured thresholds and total count for No.exceeded and max bandwidth and No.disacarded packets .
Alarms
The number of alarms on the router and their classification according to severity.
ICMP Test
ICMP is an important aspect of the error reporting process and testing to see how well a network is transmitting data,The ICMP test uses ICMP packets to determine the delay, jitter, and packet loss ratio based on the timestamps in test packets.
CPU & Memory
Focuses on displaying the CPU status and Memory of each router.

Analysis Tools

These tools are used to analyze the huge data in real time that is fetched from the database



Search by Router name or Link
Knowing the name of the router is enough to get all the information about it

Filter by Date
Fetching data from the database and displaying it according to a specific period specified by the user

Filter by Service or KPI
Fetch data where there is a threshold exceeded .


Export To Excel
Full flexibility in obtaining the results of previous tools and keeping them as an Excel file.

Draw Chart
Drawing the results of the previous tools in addition to drawing the status of a router or link depending on the KPI and the service .
